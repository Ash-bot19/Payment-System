**Step B — Create payment-backend/infra/vm-docker-compose.yml.**

This file is the VM-only subset of docker-compose.yml. It includes:
- zookeeper (identical to docker-compose.yml)
- kafka (with KAFKA_ADVERTISED_LISTENERS modified to expose VM_INTERNAL_IP on port 29092 for internal consumption + localhost:9092 for VM-local processes)
- kafka-init (identical)
- spark-feature-engine (SPARK_CHECKPOINT_DIR changed to GCS bucket path)
- validation-consumer
- scoring-consumer
- ledger-consumer

It does NOT include: redis, postgres, prometheus, grafana, webhook-service, ml-scoring-service, streamlit-dashboard, airflow-* (those run on Cloud Run or managed services).

The kafka environment block must use a placeholder for the VM internal IP:

```yaml
environment:
  KAFKA_BROKER_ID: 1
  KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181
  KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: PLAINTEXT:PLAINTEXT,PLAINTEXT_HOST:PLAINTEXT
  KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://kafka:29092,PLAINTEXT_HOST://${VM_INTERNAL_IP:-localhost}:9092
  KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
  KAFKA_AUTO_CREATE_TOPICS_ENABLE: "false"
```

Consumer services must point to Cloud SQL and Memorystore:
```yaml
environment:
  - KAFKA_BOOTSTRAP_SERVERS=kafka:29092
  - DATABASE_URL_SYNC=postgresql://payment:payment@${CLOUD_SQL_HOST}:5432/payment_db
  - REDIS_URL=redis://${REDIS_HOST}:6379/0
```

spark-feature-engine must use GCS checkpoint dir:
```yaml
environment:
  - SPARK_CHECKPOINT_DIR=gs://${GCS_BUCKET_NAME}/checkpoints
  - KAFKA_BOOTSTRAP_SERVERS=kafka:29092
  - REDIS_URL=redis://${REDIS_HOST}:6379/0
```

Replicate ALL healthcheck, restart, depends_on, and port entries verbatim from docker-compose.yml for each included service.

The file must NOT reference `../.env` (no env_file directive) since secrets come from the VM's shell environment variables, not a .env file. Consumer images pull from Artifact Registry — use image: references instead of build: sections:
```yaml
image: asia-south1-docker.pkg.dev/project-2f9d2775-493e-4e59-9b8/payment-system/validation-consumer:latest
```
(Do the same for scoring-consumer, ledger-consumer, spark-feature-engine.)

Volumes section: remove `spark_checkpoints` volume (GCS replaces it). Keep any local volumes needed.
