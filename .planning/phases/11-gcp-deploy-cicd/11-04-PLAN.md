**Step A — Implement export_to_bigquery in nightly_reconciliation.py.**

Locate the `export_to_bigquery` function (currently at line ~351). Replace the entire function body
(keeping the `@task()` decorator and docstring) with this implementation:

```python
@task()
def export_to_bigquery(last_cursor: str | None = None) -> None:
    """Export new ledger rows to BigQuery using created_at as incremental cursor.

    Reads ledger_entries from PostgreSQL where created_at > last_cursor (or
    '1970-01-01' for a full load). Loads to BigQuery table
    {GCP_PROJECT_ID}.{BIGQUERY_DATASET}.ledger_entries with WRITE_APPEND.

    Skips cleanly when GCP_PROJECT_ID is not set (local dev).
    """
    if not GCP_PROJECT_ID:
        logger.info(
            "export_to_bigquery_skipped",
            reason="GCP_PROJECT_ID_not_set",
            phase="11",
        )
        return

    import pandas as pd
    from google.cloud import bigquery

    cursor = last_cursor or "1970-01-01T00:00:00+00:00"
    engine = create_engine(DATABASE_URL_SYNC)

    df = pd.read_sql(
        """
        SELECT
            id, transaction_id, entry_type, amount_cents,
            currency, merchant_id, created_at
        FROM ledger_entries
        WHERE created_at > %(cursor)s
        ORDER BY created_at
        """,
        engine,
        params={"cursor": cursor},
    )

    if df.empty:
        logger.info("export_to_bigquery_no_rows", cursor=cursor)
        return

    client = bigquery.Client(project=GCP_PROJECT_ID)
    table_id = f"{GCP_PROJECT_ID}.{BIGQUERY_DATASET}.ledger_entries"

    job_config = bigquery.LoadJobConfig(
        write_disposition=bigquery.WriteDisposition.WRITE_APPEND,
        autodetect=True,
    )

    job = client.load_table_from_dataframe(df, table_id, job_config=job_config)
    job.result()  # blocks until complete

    logger.info(
        "export_to_bigquery_complete",
        rows_exported=len(df),
        table=table_id,
        cursor=cursor,
    )
```

Add `import pandas as pd` to the module-level imports block (before the `from google.cloud import bigquery` inside the function — keep it inside the function to avoid import error when GCP_PROJECT_ID is not set in local dev).

Do NOT add `import pandas as pd` at the top of the file — keep it inside the function body alongside `from google.cloud import bigquery`. This preserves the existing behavior where local dev (GCP_PROJECT_ID empty) exits without importing heavy GCP dependencies.

**Step C — Create the Cloud Run Job for the nightly reconciliation DAG.**

The Cloud Run Job runs the Airflow DAG as a standalone Python script (not via `airflow scheduler`).
The entrypoint calls the DAG functions directly using the Airflow standalone runner:

```bash
gcloud run jobs create nightly-reconciliation \
  --image=asia-south1-docker.pkg.dev/project-2f9d2775-493e-4e59-9b8/payment-system/airflow-dag-runner:latest \
  --region=asia-south1 \
  --project=project-2f9d2775-493e-4e59-9b8 \
  --vpc-connector=payment-connector \
  --vpc-egress=private-ranges-only \
  --set-secrets="DATABASE_URL_SYNC=DATABASE_URL_SYNC:latest,STRIPE_API_KEY=STRIPE_API_KEY:latest" \
  --set-env-vars="GCP_PROJECT_ID=project-2f9d2775-493e-4e59-9b8,BIGQUERY_DATASET=payment_analytics,KAFKA_BOOTSTRAP_SERVERS=placeholder,PYTHONPATH=/opt/airflow,GOOGLE_CLOUD_PROJECT=project-2f9d2775-493e-4e59-9b8" \
  --command="python" \
  --args="-c,from airflow.dags.nightly_reconciliation import detect_duplicates, fetch_stripe_window, compare_and_publish, persist_discrepancies, export_to_bigquery; import datetime; ds=datetime.date.today().isoformat(); duplicates=detect_duplicates(ds); intents=fetch_stripe_window(ds); msgs=compare_and_publish(intents, ds); persist_discrepancies(msgs, ds); export_to_bigquery()" \
  --memory=1Gi \
  --cpu=1 \
  --max-retries=1 \
  --task-timeout=600s
```

Note: The `--args` value above calls each DAG task function sequentially, bypassing the Airflow
scheduler overhead entirely. This is the Cloud Run Job pattern for Airflow (D-23).
