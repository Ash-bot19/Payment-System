    - `LocalExecutor` (not CeleryExecutor) — single-node local dev, no Redis/RabbitMQ needed for Airflow
    - Shared PostgreSQL instance (Airflow metadata + app tables in same DB) — acceptable for local dev
    - `AIRFLOW__CORE__FERNET_KEY` — static base64 string for local dev (NOT production)
    - `AIRFLOW__CORE__LOAD_EXAMPLES=false` — no sample DAGs cluttering UI
    - Build context is `..` (payment-backend/) so COPY commands in Dockerfile resolve correctly
    - `airflow-init` runs once to migrate DB and create admin user, then exits
    - No additional volumes needed — DAGs are baked into the image
