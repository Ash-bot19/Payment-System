**A. Create `payment-backend/scripts/seed_demo_data.py`:**

Python script using psycopg2 (no ORM). Connection via `DATABASE_URL` env var, default `postgresql://payment:payment@localhost:5432/payment_db`.

Set `conn.autocommit = False` (CRITICAL for ledger DEFERRABLE trigger).

Define 3 merchants: `merchant_a`, `merchant_b`, `merchant_c`.

Generate 15 unique transaction UUIDs (use `uuid.uuid4()`). Spread `created_at` timestamps over the last 7 days using `datetime.now(timezone.utc) - timedelta(days=random_offset)` where offset is 0-6 days with random hours.

**payment_state_log inserts (INSERT-only, check existence first via event_id):**
For each transaction, insert state transition rows simulating the full lifecycle:
- 8 SETTLED transactions (3 for merchant_a, 3 for merchant_b, 2 for merchant_c):
  Each gets 5 rows: INITIATED, VALIDATED, SCORING, AUTHORIZED, SETTLED
- 4 FLAGGED transactions (2 for merchant_a, 1 for merchant_b, 1 for merchant_c):
  Each gets 4 rows: INITIATED, VALIDATED, SCORING, FLAGGED
- 3 AUTHORIZED transactions (1 per merchant):
  Each gets 4 rows: INITIATED, VALIDATED, SCORING, AUTHORIZED

Columns per row: `transaction_id`, `from_state` (None for INITIATED, previous state otherwise), `to_state`, `event_id` (format: `seed_evt_{tx_uuid}_{state}`), `merchant_id`, `created_at` (incrementing by seconds within a transaction).

Idempotency guard: Before inserting, check `SELECT COUNT(*) FROM payment_state_log WHERE event_id = %s`. Skip if exists.

**ledger_entries inserts (DEBIT+CREDIT in same transaction):**
Only for the 8 SETTLED transactions. For each:
- amount_cents: vary between 1000 and 50000 (10.00 to 500.00 USD)
- Insert DEBIT row: `(transaction_id, amount_cents, 'DEBIT', merchant_id, 'USD', f'seed_evt_{tx_uuid}_debit')`
- Insert CREDIT row: `(transaction_id, amount_cents, 'CREDIT', merchant_id, 'USD', f'seed_evt_{tx_uuid}_credit')`
- Both in same `with conn:` / `with conn.cursor() as cur:` block, then `conn.commit()` (trigger fires at commit, sees balance = 0)

Idempotency: Check `SELECT COUNT(*) FROM ledger_entries WHERE source_event_id = %s` before each pair.

**reconciliation_discrepancies inserts:**
5 rows:
- 2 MISSING_INTERNALLY: transaction_id=new UUIDs, stripe_payment_intent_id=`pi_seed_missing_{n}`, internal_amount_cents=None, stripe_amount_cents=random 1000-5000, diff_cents=None, merchant_id=merchant_a/merchant_b, run_date=date.today()-timedelta(days=1), stripe_created_at=datetime with offset
- 2 AMOUNT_MISMATCH: transaction_id from SETTLED txns, stripe_payment_intent_id=`pi_seed_mismatch_{n}`, internal_amount_cents=the seeded amount, stripe_amount_cents=internal+random_diff, diff_cents=stripe-internal, merchant_id matching, run_date=today-1
- 1 DUPLICATE_LEDGER: transaction_id from SETTLED txn, stripe_payment_intent_id=`pi_seed_dup_1`, internal_amount_cents=amount, stripe_amount_cents=amount, diff_cents=0, merchant_id matching, run_date=today-1

Idempotency: Check `SELECT COUNT(*) FROM reconciliation_discrepancies WHERE transaction_id = %s AND discrepancy_type = %s`.

Use structlog for logging (not print). Log summary at end: "Seeded {n} state_log rows, {n} ledger rows, {n} discrepancy rows".

Script entry point: `if __name__ == "__main__": main()`.
