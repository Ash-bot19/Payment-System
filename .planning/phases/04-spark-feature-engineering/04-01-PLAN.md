       compute_hour_of_day(received_at_str: str) -> int
         Parse ISO timestamp string via datetime.fromisoformat(), return the .hour attribute (integer 0-23).

       compute_weekend_flag(received_at_str: str) -> int
         Parse ISO timestamp via datetime.fromisoformat(), return 1 if .weekday() is 5 (Saturday) or 6 (Sunday), else 0.

       compute_amount_cents_log(amount_cents: int) -> float
         Return math.log1p(float(amount_cents)).

       update_merchant_stats_and_zscore(r_client: redis.Redis, merchant_id: str, amount_cents: int) -> float
         Implements Welford's online algorithm per Research Pattern 7.
         Redis key format: merchant:stats:{merchant_id} as a Hash with fields count (int), mean (float), M2 (float).
         Steps: read existing stats via r_client.hgetall(key). Parse count/mean/m2 from returned dict (default to 0/0.0/0.0).
         Welford update: count += 1; delta = amount_cents - mean; mean += delta / count; delta2 = amount_cents - mean; m2 += delta * delta2.
         Write updated stats: r_client.hset(key, mapping={"count": count, "mean": mean, "M2": m2}).
         If count < 2: return 0.0.
         Else: variance = m2 / (count - 1); stddev = math.sqrt(variance) if variance > 0 else 1.0; return (amount_cents - mean) / stddev.

       Imports: math, redis, structlog, datetime (from datetime import datetime).
       All logging via structlog.get_logger() -- never use print().

    1. Create payment-backend/spark/redis_sink.py with the following structure:

       Imports: os, redis, structlog, and from spark.feature_functions import update_merchant_stats_and_zscore.

       Module-level constants:
         REDIS_URL = os.getenv("REDIS_URL", "redis://localhost:6379/0")
         FEATURE_TTL = 3600
         DEVICE_SEEN_TTL = 300

       create_redis_client() -> redis.Redis:
         Return redis.Redis.from_url(REDIS_URL, decode_responses=True).

       write_features_to_redis(batch_df, epoch_id):
         log = structlog.get_logger()
         rows = batch_df.collect()
         If not rows: log.info("empty_batch", epoch_id=epoch_id) and return.
         r = create_redis_client()
         pipe = r.pipeline()
         For each row in rows:
           a. merchant_risk = float(r.hget(f"merchant:risk:{row.merchant_id}", "score") or 0.5)
           b. device_key = f"device_seen:{row.stripe_customer_id}"
              last_merchant = r.get(device_key)
              device_switch = 1 if (last_merchant and last_merchant != row.merchant_id) else 0
           c. amount_zscore = update_merchant_stats_and_zscore(r, row.merchant_id, row.amount_cents)
           d. pipe.hset(f"feat:{row.event_id}", mapping={
                "tx_velocity_1m": getattr(row, "tx_velocity_1m", 0),
                "tx_velocity_5m": getattr(row, "tx_velocity_5m", 0),
                "amount_zscore": amount_zscore,
                "merchant_risk_score": merchant_risk,
                "device_switch_flag": device_switch,
                "hour_of_day": row.hour_of_day,
                "weekend_flag": row.weekend_flag,
                "amount_cents_log": row.amount_cents_log,
              })
           e. pipe.expire(f"feat:{row.event_id}", FEATURE_TTL)
           f. pipe.set(device_key, row.merchant_id, ex=DEVICE_SEEN_TTL)
         pipe.execute()   # CRITICAL -- must be called or writes are silently lost
         log.info("features_written", epoch_id=epoch_id, row_count=len(rows))
