    **`_write_ledger_entries(self, event_id, amount_cents, merchant_id, currency, source_event_id) -> None`:**
    Write exactly 2 rows (DEBIT + CREDIT) in a single transaction:
    ```python
    with self._engine.begin() as conn:
        conn.execute(
            insert(LedgerEntry.__table__).values(
                transaction_id=event_id,
                amount_cents=amount_cents,
                entry_type="DEBIT",
                merchant_id=merchant_id,
                currency=currency,
                source_event_id=source_event_id,
            )
        )
        conn.execute(
            insert(LedgerEntry.__table__).values(
                transaction_id=event_id,
                amount_cents=amount_cents,
                entry_type="CREDIT",
                merchant_id=merchant_id,
                currency=currency,
                source_event_id=source_event_id,
            )
        )
    ```
    Both inserts share the same `engine.begin()` context so the DEFERRABLE balance trigger fires at commit.

    **`_process_message(self, msg: Message) -> None`:**
    Processing order:
    1. Decode JSON from `msg.value()`
    2. Extract fields: `event_id`, `amount_cents`, `merchant_id`, `currency` (default "usd"), `source_event_id`
    3. Validate required fields (event_id, amount_cents must be int, merchant_id must exist). On validation error: DLQ with `LEDGER_WRITE_FAIL`, commit offset, return.
    4. Idempotency check: if `_is_duplicate_settled(event_id)`, log duplicate, commit offset, return.
    5. Try: `_write_ledger_entries(...)` then `_state_machine.write_transition(transaction_id=event_id, event_id=event_id, merchant_id=merchant_id, from_state=PaymentState.AUTHORIZED, to_state=PaymentState.SETTLED)`
    6. On `sqlalchemy.exc.IntegrityError` or `sqlalchemy.exc.InternalError` (trigger violation): DLQ with `LEDGER_WRITE_FAIL`, commit offset, return. These are non-retryable constraint failures.
    7. On `sqlalchemy.exc.OperationalError` (connection failure, DB down): RE-RAISE. Do NOT DLQ. Docker restart + Kafka replay handles systemic DB outages (per D-08, D-10).
    8. Commit offset: `consumer.store_offsets(msg)` then `consumer.commit()`
    9. Increment Prometheus counters.

    **Error handling distinction (critical per D-08, D-09, D-10):**
    - `IntegrityError` / `InternalError` = constraint violation = DLQ (non-retryable)
    - `OperationalError` = DB down = CRASH (retryable via Docker restart)
    - `json.JSONDecodeError` / `KeyError` / `TypeError` = malformed payload = DLQ
    - Any other unhandled exception = CRASH (same as scoring_consumer)

    **`run`, `_shutdown`, `_cleanup`, `_start_health_server`, `_HealthHandler`:** Copy pattern from scoring_consumer exactly. Health on port 8004.

    **`if __name__ == "__main__":` block** at bottom for direct execution.
