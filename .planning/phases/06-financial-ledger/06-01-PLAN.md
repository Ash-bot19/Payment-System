      ```sql
      CREATE OR REPLACE FUNCTION enforce_ledger_balance()
      RETURNS TRIGGER AS $$
      DECLARE
        balance BIGINT;
      BEGIN
        SELECT COALESCE(
          SUM(CASE WHEN entry_type = 'CREDIT' THEN amount_cents ELSE 0 END) -
          SUM(CASE WHEN entry_type = 'DEBIT' THEN amount_cents ELSE 0 END),
          0
        ) INTO balance
        FROM ledger_entries
        WHERE transaction_id = NEW.transaction_id;

        IF balance != 0 THEN
          RAISE EXCEPTION 'Ledger imbalance for transaction_id %: balance = % (must be 0)', NEW.transaction_id, balance;
        END IF;

        RETURN NULL;
      END;
      $$ LANGUAGE plpgsql;
      ```
    - Constraint trigger fires AFTER INSERT on `ledger_entries` as a DEFERRED constraint trigger:
      ```sql
      CREATE CONSTRAINT TRIGGER check_ledger_balance
        AFTER INSERT ON ledger_entries
        DEFERRABLE INITIALLY DEFERRED
        FOR EACH ROW EXECUTE FUNCTION enforce_ledger_balance();
      ```
      DEFERRABLE INITIALLY DEFERRED ensures the check runs at transaction commit, not after each individual INSERT (so both DEBIT and CREDIT can be inserted within one transaction before the balance is checked).
    - `downgrade()` drops all triggers, functions, indexes, table
