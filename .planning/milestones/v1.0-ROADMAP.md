## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**

- `confluent-kafka` over `aiokafka` — C librdkafka binding, better production track record
- Kafka key = `stripe_event_id` — ordering guarantee: all events for same payment → same partition
- Redis idempotency key set AFTER Kafka publish — avoids silent event loss on publish failure
- `KAFKA_AUTO_CREATE_TOPICS_ENABLE: false` — prevents wrong partition configs on auto-created topics
- Prometheus instrumented from M1 — zero retroactive cost; every future milestone gets metrics free
- FastAPI lifespan pattern — `@app.on_event` deprecated in 0.115+
- `cub zk-ready` for Zookeeper health check — `nc` not available in Confluent images

**Issues Resolved:**

- Zookeeper health check: `nc` not available → switched to `cub zk-ready localhost:2181 30`
- Stripe SDK v9+ exception namespace: `stripe.errors` (plural) → `stripe.error.SignatureVerificationError` (singular)
- `requirements.txt` overwritten by linter tool — restored manually; added to Known Gotchas
- Docker container name conflicts on re-run — documented `docker rm -f` protocol
- Git not initialized before architecture review — added to Session Protocol

**Issues Deferred:** None

**Technical Debt Incurred:**

- Kafka producer uses synchronous `flush(timeout=5)` per message — acceptable at M1; async batching may be needed at higher throughput
- No integration tests yet (only unit tests with mocked Redis/Kafka) — M2 will add integration test layer
- PostgreSQL provisioned but no Alembic migrations yet — M2 adds payment_state_log table
