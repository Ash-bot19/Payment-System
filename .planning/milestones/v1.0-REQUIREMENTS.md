# Requirements: Payment System — v1.0

**Defined:** 2026-03-22
**Core Value:** Every payment event is reliably ingested, deduplicated, scored for fraud risk, and recorded in an auditable double-entry ledger with no data loss.

## v1.0 Requirements

### Infrastructure

- [x] **INFRA-01**: All services (Kafka, Redis, PostgreSQL, Prometheus, Grafana) start with a single `docker-compose up -d`
- [x] **INFRA-02**: All 7 Kafka topics provisioned with locked names and correct partition config on first compose-up
- [x] **INFRA-03**: FastAPI webhook service containerised and reachable at localhost:8000

### Ingestion

- [x] **INGEST-01**: POST /webhook verifies Stripe HMAC signature and rejects invalid signatures with 400
- [x] **INGEST-02**: Duplicate Stripe events (same event_id + type) return 200 immediately and skip Kafka publish
- [x] **INGEST-03**: Valid new events are published to `payment.webhook.received` Kafka topic with stripe_event_id as partition key
- [x] **INGEST-04**: Redis idempotency key set atomically (Lua SET NX EX 86400) after successful Kafka publish

### Observability

- [x] **OBS-01**: All HTTP routes instrumented with Prometheus metrics (latency, count, errors)
- [x] **OBS-02**: All log events use structlog with bound context (stripe_event_id, event_type)

### Quality

- [x] **QUAL-01**: 5 unit tests covering happy path, duplicate skip, and invalid signature cases — all passing
- [x] **QUAL-02**: No hardcoded credentials — all secrets via python-dotenv
